Relational Structure

Database ERD

The current relational structure of the bookstore platform is shown below in a clean entity relationship diagram.

erDiagram
    profiles {
        uuid id PK
        text full_name
        text email
        text address
        text avatar_url
        text bio
        text role
        timestamp created_at
        timestamp updated_at
    }

    categories {
        bigint id PK
        text name
        text slug
        timestamp created_at
    }

    authors {
        bigint id PK
        text name
        text nationality
        int birth_year
        text biography
        timestamp created_at
    }

    books {
        bigint id PK
        bigint author_id FK
        bigint category_id FK
        text title
        text isbn
        int publish_year
        int pages
        text language
        text format_type
        text short_description
        text description
        text cover_url
        boolean is_published
        timestamp created_at
        timestamp updated_at
    }

    favorites {
        bigint id PK
        uuid user_id FK
        bigint book_id FK
        timestamp created_at
    }

    profiles ||--o{ favorites : saves
    books ||--o{ favorites : appears_in
    authors ||--o{ books : writes
    categories ||--o{ books : groups

Schema Notes for Presentation

profiles

Stores the public user profile fields such as full name, email, address, avatar, biography, and role.

authors

Contains writer information such as name, nationality, birth year, and biography.

categories

Separates books into searchable genres or topic groups for easier catalog browsing.

books

The main catalog table that links each book to one author and one category.

favorites

Keeps the relation between a user profile and the books the user has saved as favorites.
