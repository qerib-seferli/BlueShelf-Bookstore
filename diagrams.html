Database ERD

The database structure is shown visually in an ERD style similar to a formal relational schema presentation.

erDiagram
    profiles {{
        uuid id PK
        text full_name
        text email
        text address
        text avatar_url
        text bio
        text role
        timestamp created_at
        timestamp updated_at
    }}

    categories {{
        bigint id PK
        text name
        text slug
        timestamp created_at
    }}

    authors {{
        bigint id PK
        text name
        text nationality
        int birth_year
        text biography
        timestamp created_at
    }}

    books {{
        bigint id PK
        bigint author_id FK
        bigint category_id FK
        text title
        text isbn
        int publish_year
        int pages
        text language
        text format_type
        text short_description
        text description
        text cover_url
        boolean is_published
        timestamp created_at
        timestamp updated_at
    }}

    favorites {{
        bigint id PK
        uuid user_id FK
        bigint book_id FK
        timestamp created_at
    }}

    profiles ||--o{{ favorites : saves
    books ||--o{{ favorites : appears_in
    authors ||--o{{ books : writes
    categories ||--o{{ books : groups

Schema Notes for Presentation

profiles keeps authenticated user data and admin role control.
authors stores biography and identity details for each writer.
categories separates books into searchable genres.
books is the main catalog table linked to authors and categories.
favorites creates a many-to-one user save system for selected books.
