Relational Structure

Database ERD

The bookstore schema is presented below in a structured relational format inspired by formal academic ERD layouts.

Public Tables: profiles, authors, categories, books, favorites
One-to-many relations are visualized below 
erDiagram
    profiles {
        uuid id PK
        text full_name
        text email
        text address
        text avatar_url
        text bio
        text role
        timestamp created_at
        timestamp updated_at
    }

    authors {
        bigint id PK
        text name
        text nationality
        int birth_year
        text biography
        timestamp created_at
    }

    categories {
        bigint id PK
        text name
        text slug
        timestamp created_at
    }

    books {
        bigint id PK
        bigint author_id FK
        bigint category_id FK
        text title
        text isbn
        int publish_year
        int pages
        text language
        text format_type
        text short_description
        text description
        text cover_url
        boolean is_published
        timestamp created_at
        timestamp updated_at
    }

    favorites {
        bigint id PK
        uuid user_id FK
        bigint book_id FK
        timestamp created_at
    }

    authors ||--o{ books : writes
    categories ||--o{ books : groups
    profiles ||--o{ favorites : saves
    books ||--o{ favorites : appears_in

Schema Notes for Presentation

profiles

Stores each authenticated user’s public information such as full name, email, address, avatar, biography, and role.

authors

Stores author identity details and biography information used on author and book detail views.

categories

Defines genres and catalog groups so books can be filtered and searched by category.

books

The main catalog table. Each book connects to one author and one category and contains all display content.

favorites

Creates the saved-books relationship between a user profile and the books selected by that user.
